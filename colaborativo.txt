<h1><a name="1"></a>1. Introducción</h1>
<ul>
<li>HTML<a name="ll1"></a><a href="#n1"><sup>1</sup></a> es un lenguaje universal, que funciona en cualquier plataforma (Windows, Macintosh, Unix, OS/2, etc.) y 
con cualquier navegador o browser (Netscape, Internet Explorer, Mozilla Firefox, etc.). El precio que paga por su universalidad es su poca sofisticación, puesto 
que no es otra cosa que el viejo formato ASCII<a name="ll2"></a><a href="#n2"><sup>2</sup></a> (y con 7 bits en lugar de ocho, por lo que ni tan siquiera tiene 
acentos ni otros caracteres especiales).</li>
<li>Por tanto, para que un documento HTML sea algo más que simples caracteres básicos, debe contener, además de dicho texto, una serie de instrucciones para el 
browser que lo va a reproducir: estas intrucciones se denominan etiquetas o tags y se distinguen del texto porque van entre guiones 
(<span class="B">&lt; &gt;</span>)<a name="ll3"></a><a href="#n3"><sup>3</sup></a>. Estas etiquetas contienen todo el resto de la información de la página web.</li>
<li>Por tanto, si un documento HTML no es más que texto ASCII (parte sin guiones y parte entre guiones: las etiquetas), cualquier documento web, hasta el más 
sofisticado, puede escribirse directamente desde un sencillo programa básico de texto, como el Cuaderno de Notas (<i>Notepad</i>) de Windows, por ejemplo.</li>
<li>Sin embargo, escribir un documento complejo de esta manera exige un conocimiento exhaustivo de las numerosísimas etiquetas existentes y sus normas de 
utilización; para evitar esta dificultad, hay una serie de programas denominados comúnmente editores de HTML (desde el Composer al Dreamweaver, pasando por tantos 
otros), que permiten al usuario componer una página (es decir, generar etiquetas HTML) desde un interfaz más o menos parecido al de un procesador de textos.</li>
<li>A la hora de crear una página web, y aunque utilicemos un editor de HTML, es casi imprescindible entender la disposición del etiquetado de un documento y 
manejar siquiera las etiquetas más elementales. Son varias las razones que pueden aducirse para ello:
<ol>
<li>Los editores de HTML no son siempre herramientas perfectas, sobre todo cuando ha habido muchas modificaciones durante el proceso de creación. Con bastante 
frecuencia se hará necesario consultar el código completo, con las etiquetas HTML (lo que se suele denominar <b>código fuente</b> o <b>código madre</b>) para 
corregir las disfunciones.</li>
<li>Tal vez la mejor manera de aprender a diseñar páginas web es observar las páginas de los demás, cuyos códigos fuente están siempre disponibles para ser 
imitados (o incluso copiados, técnicamente, al menos). Pero, para ello, es necesario entenderlos.</li>
<li>Los mismos editores HTML manejan términos propios del lenguaje HTML (<i>heading</i>, <i>cellspacing</i>, etc.), que es preciso comprender.</li>
<li>Es necesario un buen conocimiento de HTML para insertar otros códigos más potentes, como los de Javascript, por ejemplo.</li>
</ol></li>
<li>El objetivo de este documento es aprender a componer una sencilla página web desde el más básico de los programas de texto disponibles en Windows: el Cuaderno 
de Notas.<a name="ll4"></a><a href="#n4"><sup>4</sup></a> De esta manera, seremos capaces de entender y manejar los códigos fuente de las páginas más complejas.</li>
</ul>

<a name="2"><h1>2. Esquema mínimo de un documento HTML</h1></a>
<ul>
<li>Un documento HTML comienza siempre con la etiqueta <span class="B">&lt;HTML&gt;</span><a name="ll5"></a><a href="#n5"><sup>5</sup></a>, que indica que el 
documento en cuestión está construido con dicho lenguaje.</li>
<li>La mayoría de las etiquetas son pareadas, es decir, <span class="B">&lt;...&gt;</span> corresponde al principio de la acción y 
<span class="B">&lt;/...&gt;</span> indica el fin de dicha acción.</li>
<li>Por tanto, una página web estará siempre contenida entre las etiquetas <span class="B">&lt;HTML&gt;</span> y <span class="B">&lt;/HTML&gt;</span>.</li>
<li>Por otra parte, todo documento HTML consta de dos partes: la cabecera (<b>head</b>) y el cuerpo del documento (<b>body</b>).
	<ol>
	<li>La cabecera contiene básicamente información destinada al browser (o navegador), que queda oculta al usuario. Su etiqueta (pareada) es 
	<span class="B">&lt;HEAD&gt;</span>.</li>
	<li>El cuerpo es el documento que ve el usuario. Su etiqueta (pareada) es <span class="B">&lt;BODY&gt;</span>.</li>
	</ol>
	</li>
<li>Ya estamos en condiciones de componer nuestra primera página web. Basta con escribir las siguientes etiquetas en nuestro editor de texto y salvar el resultado 
con la extensión <b>*.HTML</b> o <b>*.HTM</b>:</li>
</ul>

<a name="3"></a><h1>3. El texto básico de un documento HTML</h1>
<ul>
<li>El texto básico de un documento HTML puede escribirse sin etiquetas. Cada navegador lo visualiza entonces con el tipo y tamaño de caracteres escogidos en su 
configuración por defecto, e introduce un salto de línea cuando los caracteres alcanzan el borde de la ventana.</li>
<li>Para dominar las cuestiones de formato (de caracteres, de párrafo, etc.) es necesario utilizar las etiquetas correspondientes.</li></ul>

<a name="31"></a><h2>3.1. La etiqueta básica</h2>
<ul>
<li>La etiqueta básica, que controla el tipo de fuente utilizado, es <span class="B">&lt;FONT&gt;</span>.</li>
<li><span class="B">&lt;FONT&gt;</span> es una etiqueta pareada. Por tanto, afecta a los caracteres introducidos entre <span class="B">&lt;FONT&gt;</span> y 
<span class="B">&lt;/FONT&gt;</span>. Pero sin más especificaciones, la etiqueta aún no sirve para nada.</li>
<li>Se puede especificar <b>el tipo de letra</b> (es decir, la fuente de caracteres) añadiéndolo a la etiqueta, de la siguiente manera (en este caso, la fuente 
escogida es Arial): <span class="B">&lt;FONT FACE="Arial"&gt;...&lt;/FONT&gt;</span>.</li>
<li>Se puede especificar <b>el color de la fuente</b> con la etiqueta <span class="B">&lt;FONT COLOR="?"&gt;...&lt;/FONT&gt;</span>. Para especificar los 
colores, conviene saber al menos lo siguiente:
	<ol>
	<li>Cada color posee su correspondiente código.</li>
	<li>Hay dos códigos: hexadecimal –lo más habitual– o RGB. En ambos casos se trata de expresar las proporciones de tres colores básicos: rojo, 
	verde y azul.</li>
	<li>Los valores hexadecimales de los tres colores se expresan de la siguiente manera: 
		<ol>
		<li>Rojo: color:#ff0000;</li>
		<li>Verde: color:#00ff00;</li>
		<li>Azul: color:#0000ff;</li>
		</ol> 
 	</li><li>Los mismos colores, expresados con los valores RGB (red, green, blue), son los siguientes: 
		<ol>
		<li>Rojo: color:rgb(255,0,0);</li>
		<li>Verde: color:rgb(0,255,0);</li>
		<li>Azul: color:rgb(0,0,255);</li>
		</ol>
	</li><li>Todos los colores se expresan mediante combinaciones de estos tres colores básicos. Existen muchos programas que proporcionan los valores de todos los 
	colores que el usuario puede componer. También en internet existen muchas páginas con los 
	<a href="http://html-color-codes.info/codigos-de-colores-hexadecimales/" target="_blank">Códigos de colores</a>.</li> 
	<li>Los colores más básicos pueden especificarse sin código, simplemente escribiendo la palabra correspondiente (por supuesto en inglés). Por ejemplo: 
	<span class="B">&lt;FONT COLOR="blue"&gt;...&lt;/FONT&gt;</span>, <span class="B">&lt;FONT COLOR="red"&gt;...&lt;/FONT&gt;</span>, etc.</li>
	</ol>
</li>
<li>Se puede especificar <b>el tamaño de la fuente</b> con la etiqueta <span class="B">&lt;FONT SIZE="?"&gt;...&lt;/FONT&gt;</span>. El valor del tamaño ("?") 
deseado puede suministrarse de manera absoluta (un número del 1 al 7) o relativa (+1, +2, -1, -2, etc., teniendo en cuenta que el valor por defecto es 
3).<a name="ll6"></a><a href="#n6"><sup>6</sup></a></li>
<li>Todas estas etiquetas se pueden combinar, de modo que con una sola se controla, por ejemplo, el tipo, el tamaño y el color de los caracteres: 
<span class="B">&lt;FONT FACE="Arial" COLOR="#0000FF" SIZE="5"&gt;...&lt;/FONT&gt;</span>.</li>
</ul>

<p class="abstract">Con esta etiqueta hemos visto ya la sintaxis básica de las etiquetas HTML, que podíamos formalizar así: &lt;ETIQUETA [ATRIBUTO="valor"]&gt;.</p> 
<p class="abstract">Lo anterior quiere decir que una etiqueta HTML (por ejemplo, FONT) puede llevar uno o varios atributos (FACE, COLOR, SIZE), cuyos valores se 
colocan entrecomillados tras el signo igual (="arial", ="#0000FF", etc.).</p>
<p class="abstract">Esto será válido para todas las etiquetas HTML que veamos a partir de este momento.</p>

<li>Volvamos a nuestro documento web y practiquemos con las posibilidades de la etiqueta <span class="B">&lt;FONT&gt;</span>. Por ejemplo:</li></ul>

</tbody></table></div>

<p class="abstract">Hay otra cuestión importante que debe saberse sobre la etiqueta &lt;FONT&gt;: es la etiqueta más básica, y conviene empezar a aprender por 
ella, pero está en desuso y desaparecerá en las versiones futuras de HTML. En su lugar, existen otras etiquetas más potentes, que iremos viendo en su momento.</p>

<a name="32"></a><h2>3.2. Etiquetas de formato de caracteres</h2>
<ul>
<li>A partir de este momento, se trata simplemente de añadir más etiquetas a nuestra aún pequeña colección, de modo que vayamos teniendo cada vez más control 
sobre el formato de los caracteres. La tabla siguiente muestra las más comunes:</li></ul>
<div align="center"><table cellspacing="0" cellpadding="2">
<tbody><tr class="filatitulo"><td>Código</td><td>Función</td></tr>
<tr class="filazul"><td><span class="B">&lt;B&gt;...&lt;/B&gt;</span></td><td>Negrita</td></tr>
<tr class="filagris"><td><span class="B">&lt;I&gt;...&lt;/I&gt;</span></td><td>Cursiva</td></tr>
<tr class="filazul"><td><span class="B">&lt;U&gt;...&lt;/U&gt;</span></td><td>Subrayado</td></tr>
<tr class="filagris"><td><span class="B">&lt;SUB&gt;...&lt;/SUB&gt;</span></td><td>Subíndice</td></tr>
<tr class="filazul"><td><span class="B">&lt;SUP&gt;...&lt;/SUP&gt;</span>&nbsp;&nbsp;</td><td>Superíndice</td></tr>
</tbody></table></div>
<ul>
<li>Hay que realizar algunos comentarios sobre las etiquetas precedentes:
<ol>
<li>Existe otra etiqueta similar a <span class="B">&lt;B&gt;</span>, menos utilizada porque depende de las opciones de configuración del navegador: 
<span class="B">&lt;STRONG&gt;...&lt;/STRONG&gt;</span>.</li>
<li>Existe otra etiqueta similar a <span class="B">&lt;I&gt;</span>, menos utilizada porque depende de las opciones de configuración del navegador: 
<span class="B">&lt;EM&gt;...&lt;/EM&gt;</span>.</li>
<li>Conviene tener cuidado a la hora de manejar el subrayado, puesto que éste se utiliza convencionalmente para indicar los hiperenlaces, y podría por tanto 
resultar confuso para el usuario.</li>
</ol></li>
<li>Es normal utilizar varias etiquetas para un mismo elemento de texto. En ese caso, hay que tener cuidado para encajarlas correctamente. Por ejemplo:<br>
<span class="B">&lt;FONT FACE="Courier"&gt;&lt;B&gt;&lt;I&gt;...&lt;/I&gt;&lt;/B&gt;&lt;/FONT&gt;</span> es correcto, pero<br>
<span class="B">&lt;FONT FACE="Courier"&gt;&lt;B&gt;&lt;I&gt;...&lt;/FONT&gt;&lt;B&gt;&lt;/I&gt;</span> es incorrecto.</li>
</ul>

<a name="33"></a><h2>3.3. Otras etiquetas de caracteres</h2>
<ul>
<li>Hay muchas otras etiquetas que afectan a los caracteres. Vamos a presentar varias más, que deben manejarse con cierta precaución:
<ol>
<li>La etiqueta <span class="B">&lt;PRE&gt;...&lt;/PRE&gt;</span> indica que el texto afectado está preformateado, de modo que el navegador sí tiene en cuenta 
todos los espacios y saltos de línea definidos en la pantalla.</li>
<li>La etiqueta <span class="B">&lt;BLINK&gt;...&lt;/BLINK&gt;</span> indica texto parpadeante. Sólo funciona con Netscape.</li>
<li>La etiqueta <span class="B">&lt;S&gt;...&lt;/S&gt;</span> genera texto tachado.</li>
<li>La etiqueta <span class="B">&lt;TT&gt;...&lt;/TT&gt;</span> genera texto similar al de una máquina de escribir (fuente courier de paso fijo).</li>
<li>La etiqueta <span class="B">&lt;BIG&gt;...&lt;/BIG&gt;</span> aumenta el tamaño de letra (equivale a <span class="B">&lt;FONT SIZE="+1"&gt;</span>).</li>
<li>La etiqueta <span class="B">&lt;SMALL&gt;...&lt;/SMALL&gt;</span> disminuye el tamaño de letra (equivale a <span class="B">&lt;FONT SIZE="-1"&gt;</span>).</li>
</ol></li></ul>

<a name="4"></a><h1>4. Los párrafos de un documento HTML</h1>
<p class="abstract">En el apartado anterior hemos aprendido a controlar la apariencia de los caracteres. (Recuerda, de todas maneras, que la etiqueta &lt;FONT&gt; 
está en desuso.) Vamos ahora a estudiar las etiquetas fundamentales que controlan la organización de los párrafos en un texto más complejo.</p>
<ul>
<li>Ya conocemos un tag que afecta a las líneas: <span class="B">&lt;BR&gt;</span>. Si deseamos que una línea no se parta al llegar al final de la ventana del 
navegador, debemos colocar al comienzo la etiqueta <span class="B">&lt;NOBR&gt;</span>, y el navegador colocará entonces un <i>scroll</i> o barra de desplazamiento 
horizonal.</li>
<li>Por supuesto, si queremos que haya más líneas, al final sí deberá aparecer <span class="B">&lt;BR&gt;</span>.</li>
</ul>

<a name="41"></a><h2>4.1. Los párrafos y su alineamiento</h2>
<ul>
<li>La etiqueta <span class="B">&lt;BR&gt;</span> inserta un salto de línea, pero no un salto de párrafo.<a name="ll7"></a><a href="#n7"><sup>7</sup></a> En 
cambio, la etiqueta <span class="B">&lt;P&gt;...&lt;/P&gt;</span> inserta una salto de línea y una línea en blanco, por lo que, en la práctica, equivale a un 
nuevo párrafo.</li>
<li>Por supuesto, tanto <span class="B">&lt;BR&gt;</span> como <span class="B">&lt;P&gt;...&lt;/P&gt;</span> pueden repetirse varias veces seguidas para dejar más 
distancia entre las líneas y los párrafos.</li>
<li>De todas maneras, sólo podemos separar los párrafos con líneas completas (una, dos, etc.), pero no podemos controlar el espacio entre los párrafos con la 
precisión que permite, por ejemplo, el programa Word (con las opciones de espaciado anterior y posterior).<a name="ll8"></a><a href="#n8"><sup>8</sup></a></li>
<li>Un párrafo puede alinearse introduciendo los siguientes atributos en la etiqueta habitual:</li>
</ul>
<div align="center"><table cellspacing="0" cellpadding="2">

<tr class="filazul"><td><span class="B">&lt;P align="center"&gt;...&lt;/P&gt;&nbsp;&nbsp;</span></td><td>Párrafo centrado</td></tr>
<tr class="filagris"><td><span class="B">&lt;P align="right"&gt;...&lt;/P&gt;</span></td><td>Párrafo alienado a la derecha</td></tr>
<tr class="filazul"><td><span class="B">&lt;P align="justify"&gt;...&lt;/P&gt;</span></td><td>Párrafo justificado</td></tr>
</tbody></table></div>
<ul>
<li>Las etiquetas anteriores permiten alinear los párrafos enteros, pero siempre dejando una línea vacía delante de cada uno de ellas. ¿Hay alguna manera de 
alinear independientemente líneas o grupos de líneas? Sí, con las siguientes etiquetas:</li>
</ul>
<div align="center"><table cellspacing="0" cellpadding="2">

</tbody></table></div>

<a name="42"></a><h2>4.2. Los títulos</h2>

otra después.</li>
<li>Las etiquetas de los títulos admiten indicaciones de alineación, de la misma manera que la etiqueta de párrafo.</li>

<a name="43"></a><h2>4.3. Las listas</h2>
<ul>
<li>Las listas son elementos que, como los títulos, permiten presentar niveles jerarquizados de texto.</li>
<li>Los documentos HTML permiten dos tipos de listas:
<ol>
<li>Listas no ordenadas (<i>bullet list</i>): su etiqueta principal es <span class="B">&lt;UL&gt;...&lt;/UL&gt;</span>.</li>
<li>Listas ordenadas o numeradas (<i>numbered list</i>): su etiqueta principal es <span class="B">&lt;OL&gt;...&lt;/OL&gt;</span>.</li>
<li>Listas de definiciones (<i>definition list</i>): su etiqueta principal es <span class="B">&lt;DL&gt;...&lt;/DL&gt;</span></li>.
</ol></li>
<li>Una vez definido el tipo de lista deseada, los elementos que la componen se marcan de distinta manera según el tipo:
<ol>
<li>En las listas ordenadas (<span class="B">&lt;OL&gt;...&lt;/OL&gt;</span>) o no ordenadas (<span class="B">&lt;UL&gt;...&lt;/UL&gt;</span>), cada elemento de 
dicha lista se marca con la etiqueta <span class="B">&lt;LI&gt;...&lt;/LI&gt;</span> (<i>list item</i>).</li>
<li>En las listas de definición (<span class="B">&lt;DL&gt;...&lt;/DL&gt;</span>), se utilizan dos: cada elemento que se define se marca con
<span class="B">&lt;DT&gt;...&lt;/DT&gt;</span> y el texto de cada definición se marca con <span class="B">&lt;DD&gt;...&lt;/DD&gt;</span>. Esta etiqueta se puede 
repetir si se necesita introducir varios elementos de definición.</li>
</ol>
</li>

</ul>

<li>Las listas admiten varios parámetros, que varían según el tipo de lista:
<ol>
<li>Las tres listas admiten el parámetro <span class="B">COMPACT</span>, que hace que el navegador presente la lista de la manera más compacta posible.</li>
<li>El parámetro <span class="B">TYPE</span> funciona con las dos primeras listas. Sus valores respectivos son, lógicamente, distintos:
	<ul>
	<li>En las no numeradas, controla el tipo de viñeta: con forma de disco (<span class="B">&lt;UL type="disc"&gt;</span>, la opción por defecto), de círculo 
	(<span class="B">&lt;UL type="circle"&gt;</span>) y de cuadrado (<span class="B">&lt;UL type="square"&gt;</span>).</li>
	<li>En las numeradas, controla el tipo de numeración: con números árabes (<span class="B">&lt;OL type="1"&gt;</span>, la opción por defecto), romanos 
	(<span class="B">&lt;OL type="I"&gt;</span>), romanos en minúsculas (<span class="B">&lt;OL type="i"&gt;</span>), letras mayúsculas 
	(<span class="B">&lt;OL type="A"&gt;</span>) y minúsculas (<span class="B">&lt;OL type="a"&gt;</span>).</li>
	</ul>
</li>
<li>La etiqueta <span class="B">&lt;OL start="n"&gt;</span> indica el número por el que empieza la lista numerada.</li>
<li>En una lista numerada, se puede cambiar el número de un elemento –y consiguientemente de los posteriores– con la etiqueta 
<span class="B">&lt;LI value="n"&gt;</span>.</li> 
</ol>
</li>
</ul>

<a name="44"></a><h2>4.4. Las rayas horizontales</h2>
<ul>
<li>Las rayas horizontales son otro de los procedimientos habituales para separar los contenidos de un documento HTML. Se introducen simplemente con la etiqueta 
<span class="B">&lt;HR&gt;</span>.</li>

<ol>
<li><b>El tamaño</b>: la etiqueta <span class="B">&lt;HR&gt;</span> introduce una raya que ocupa todo el ancho de la pantalla. Pueden especificarse distintas 
anchuras añadiendo el valor deseado, expresado en pixeles (<span class="B">&lt;HR width="n"&gt;</span>) o en términos porcentuales 
(<span class="B">&lt;HR width="n%"&gt;</span>).<a name="ll9"></a><a href="#n9"><sup>9</sup></a></li>
<li><b>La alineación</b>: cuando una raya ocupa sólo una parte de la pantalla, puede especificarse su alineación por medio de las etiquetas habituales:<br>
<div align="center"><span class="B">&lt;HR align="left"&gt;<br>

<li><b>La altura</b>: se expresa en píxeles por medio de la etiqueta <span class="B">&lt;HR size="n"&gt;</span>. Puede adoptar el valor 0 (la raya más fina).</li>
<li><b>El color</b>, por medio del párametro habitual: <span class="B">&lt;HR color="?"&gt;</span>.</li>
<li><b>El sombreado</b>: la raya posee habitualmente un pequeño efecto de sombreado, que puede eliminarse con la etiqueta 
<span class="B">&lt;HR noshade&gt;</span>.<a name="ll10"></a><a href="#n10"><sup>10</sup></a></li>
</ol>
</li>

<a name="5"></a><h1>5. Los enlaces de un documento HTML</h1>
<ul>
<li>En los apartados anteriores hemos aprendido a manejar correctamente el elemento fundamental de todo documento: el texto. Sin embargo, hay otro elemento básico 
en un documento HTML, que le proporciona su mayor potencial: los hiperenlaces, que estudiaremos en este apartado.</li>
<li>Un hiperenlace es un vínculo que une un elemento del documento activo (por ejemplo, una palabra) con otro, de manera que activamos ese segundo elemento 
simplemente con un <i>click</i> del ratón en el primero.<a name="ll11"></a><a href="#n11"><sup>11</sup></a></li>
<li>La sintaxis general de la etiqueta es muy simple: <span class="B">&lt;A HREF="destino"&gt;...&lt;/A&gt;</span>. (Los puntos suspensivos representan el 
elemento enlazado.)</li>
<li>Para que el destino se abra en una ventana nueva, hay que indicar el siguiente atributo: <span class="B">&lt;A HREF="destino" target="_blank"&gt;...&lt;/A&gt;</span></li>
<li>El destino de un hiperenlace puede ser de tres tipos:
<ol>
<li><b>Enlace externo</b>: otro documento HTML situado en un ordenador remoto.</li>
<li><b>Enlace local</b>: otro documento HTML situado en el mismo ordenador.</li>
<li><b>Ancla</b>: otro lugar dentro del mismo documento HTML.</li>
</ol></li>
<li>Vamos a examinar separadamente cada uno de estos tipos.</li>
</ul>

<a name="51"></a><h2>5.1. Los enlaces externos</h2>
<ul>
<li>Desde cualquier punto de un documento HTML puede establecerse un enlace con cualquier recurso de Internet.</li>
<li>Como es sabido, los recursos más habituales de Internet son:
<ol>
<li>La WWW, es decir, las páginas web o documentos HTML. La etiqueta correspondiente es:<br> 
<div align="center"><span class="B">&lt;A HREF="http://servidor/camino.../fichero"&gt;...&lt;/A&gt;</span></div></li>
<li>El correo electrónico. La etiqueta correspondiente es:<br>
<div align="center"><span class="B">&lt;A HREF="mailto:usuario@servidor"&gt;...&lt;/A&gt;</span></div></li>
<li>La transferencia de ficheros (servidores FTP). La etiqueta correspondiente es:<br>
<div align="center"><span class="B">&lt;A HREF="ftp://servidor/camino.../fichero"&gt;...&lt;/A&gt;</span></div></li>
</ol></li>
<li>Veamos algunos ejemplos:</li></ul>

<a name="52"></a><h2>5.2. Los enlaces locales</h2>
<ul>
<li>La mayor parte de las "páginas web" son, en rigor, una colección de páginas, puesto que la organización clásica de los sitios web consiste en un conjunto 
ordenado de documentos HTML entrelazados y residentes en un mismo ordenador.</li>
<li>Dentro de un mismo ordenador, los documentos HTML pueden residir en un mismo directorio o en varios:
<ol>
<li>Cuando todos los documentos residen en el mismo directorio no hay ningún problema, y la etiqueta será, simplemente: 
<span class="B">&lt;A HREF="fichero.htm"&gt;...&lt;/A&gt;</span>.</li>
<li>Cuando los ficheros residen en diferentes directorios, hay que facilitar la dirección completa del fichero, lo que puede hacerse de manera absoluta o relativa:
<ul>
<li>Direccionamiento absoluto: se indica la trayectoria completa del archivo en cuestión:<br>
<div align="center"><span class="B">&lt;A HREF=file:///c:/directorio/subdirectorios/fichero.htm"&gt;...&lt;/A&gt;</span></div></li>
<li>Direccionamiento relativo: no se indican los nombres de los directorios. Por ejemplo:<br>
<div align="center"><span class="B">&lt;A HREF=../../..fichero.htm"&gt;...&lt;/A&gt;</span></div></li>
</ul></li>
</ol></li>
<li>Aun utilizando un direccionamiento relativo, es necesario indicar los nombres de los directorios "descendentes". Por ejemplo, en una web ordenada como en la 
figura siguiente, un enlace desde un documento situado en el directorio <b>&lt;HTML&gt;</b> a un recurso situado en el directorio <b>&lt;Imágenes&gt;</b> debería 
especificarse así:</li>
</ul>

<a name="53"></a><h2>5.3. Las anclas</h2>
<ul>
<li>Especialmente cuando el documento es largo, conviene utilizar enlaces que apunten hacia otro lugar dentro del mismo documento. Estos enlaces se denominan 
anclas, anclajes o apuntadores.</li>
<li>El procedimiento de enlace es doble:
<ol>
<li>En primer lugar, hay que establecer el punto de anclaje en el lugar del documento que vaya a ser el destino del enlace. Su etiqueta es 
<span class="B">&lt;A NAME=nombre&gt;...&lt;/A&gt;</span>.</li>
<li>Después sólo queda enlazar los caracteres deseados con el ancla anterior. Su etiqueta es <span class="B">&lt;A HREF=#nombre&gt;...&lt;/A&gt;</span>.</li>
</ol></li>
<li>Algunas observaciones sobre las anclas:
<ol>
<li>El nombre del ancla es arbitrario, pero lo lógico es que guarde alguna relación con el destino, especialmente cuando hay muchas en un mismo documento.</li>
<li>Aunque la etiqueta del ancla sea pareada, lo normal es que no incluya nada entre sus dos partes (<span class="B">&lt;A NAME=nombre&gt;...&lt;/A&gt;</span>), 
puesto que lo importante es marcar el punto de inicio del ancla, no su final.</li>
</ol></li>
<li>Vamos a practicar estas etiquetas creando una nota a pie de página como las utilizadas en este documento, que implican un doble anclaje:</li>
</ul>
